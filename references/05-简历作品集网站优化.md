# 05-简历作品集网站优化

用于阶段 3：当用户提供简历、项目列表、作品集、个人网站、案例页截图或链接，并希望转岗 AI/AIGC 产品经理时，先把材料改成“招聘方能快速识别的 AI 产品证据链”。

## 核心原则

所有求职材料都要在 30 秒内回答 5 个问题：

1. 用户投的是 AI/AIGC 产品相关岗位，不是泛设计、泛运营或泛项目岗。
2. 用户有哪些可验证的 AI 项目、Workflow、原型、作品或工具链实践。
3. 原行业经验如何迁移到 AI 产品能力。
4. 哪些项目最值得被 HR、业务负责人和面试官继续追问。
5. 下一步 CTA 是什么：下载简历、查看案例、联系、约面。

不要把“会用 AI 工具”包装成“有成熟 AI 产品经验”。优先表达为：场景拆解、任务流程、工具链选择、Prompt/Agent/RAG/Workflow 实验、输出校验、bad case 复盘、迭代计划。

## 输入清单

最少需要：

- 目标岗位或 JD：AI 产品、AIGC 产品、AI Workflow、AI 产品运营、垂类工具、ToB/ToC。
- 当前简历：PDF、Word、Markdown 或文本。
- 项目列表：项目名、目标用户、痛点、功能、AI 链路、输出、验证方式。
- 作品集或个人网站：链接、截图、页面结构、案例页。
- 约束：可公开信息、不能公开的信息、目标城市/远程、投递时间线。

信息不足时，先按现有材料诊断，并列出必须补的证据。

## 简历优化

### 30 秒首屏判断

简历首屏必须出现：

```text
[姓名/称呼]
AI 产品经理候选人 / AIGC 产品方向 / AI Workflow 产品方向
[目标方向关键词 2-4 个]
[邮箱/电话/作品集链接，可用占位符]
```

不要把第一身份写成“想转行的人”。谨慎使用：

```text
正在转行 / 希望进入 AI / AI 爱好者 / 学习中 / 设计师转 AI
```

可以改成：

```text
关注 AI 产品经理 / AIGC Workflow 方向，具备 [年限] 年 [原行业] 项目经验与 AI 工具链实践。
```

### Profile 公式

```text
[年限] 年 [原行业/岗位] 经验，关注 [目标 AI 产品方向]。
擅长从 [真实场景/用户任务/业务流程] 中拆解 [痛点/约束/目标]，并转化为 [需求、信息架构、原型、Workflow、评估标准]。
已完成 [项目数量/作品类型]，覆盖 [AI 工具链/Prompt/AIGC/RAG/Agent/评测/内容质量/流程提效] 等方向。
```

没有真实商业上线经验时，不写“负责 AI 产品从 0 到 1 上线”。改写为：

```text
完成个人/练习/原型级 AI 产品探索，覆盖需求拆解、工作流设计、输出校验与迭代复盘。
```

### 一页简历优先

一页版适合初级 AI PM、AIGC 产品专员、AI 产品运营、Workflow 产品助理、设计/内容工具方向。

推荐比例：

```text
顶部身份区：15%
Profile：15%
Core Skills：15%
AI Product Projects：35%
原行业/交付经历：15%
Education / Tools / Portfolio：5%
```

重点：AI Product Projects 是最大模块；原行业经历保留，但不能抢走第一身份。

### 项目选择

简历只放 2-3 个最强项目，其它项目折叠为补充探索。

优先放：

- 能证明用户场景和痛点的项目。
- 有明确 AI Workflow / Prompt / 工具链 / 评估标准的项目。
- 能被 JD 关键词复用的项目。
- 能讲清楚边界和下一步迭代的项目。

谨慎展开：

- 涉及隐私、健康、金融、医疗等合规风险但没有专业背书的项目。
- 只有界面或效果展示，没有用户、问题、流程、验证的项目。
- 没有完整链路的写作/内容/生成类小实验。

### 项目 bullet 公式

```text
- 面向 [目标用户] 的 [场景/任务]，识别 [核心痛点]，设计 [AI 产品原型/Workflow/功能模块]。
- 拆解 [输入、处理、输出、校验] 链路，沉淀 [Prompt 模板/评估维度/SOP/工具链]。
- 使用 [工具/API/平台] 完成 [原型/网站/流程验证]，输出 [结果样例/页面/报告/任务流]。
- 通过 [用户反馈/效率对比/bad case/人工评审] 复盘效果，下一步迭代为 [优化点]。
```

## 作品集优化

### 作品集定位

作品集不要只像视觉展示页，要像 AI 产品案例库。首页或目录页建议用：

```text
AI 产品原型 / AIGC Workflow / AI 工具链探索
```

每张项目卡片用 4 个字段：

```text
用户 User
痛点 Problem
AI 方案 AI Flow
能力 Proof
```

### 项目页统一模板

每个 AI 项目页至少包含：

1. 项目一句话。
2. 目标用户。
3. 核心痛点。
4. 产品目标。
5. 功能模块。
6. AI 工具链 / Workflow。
7. 关键页面或原型。
8. 输出结果示例。
9. 质量评估标准。
10. bad case 与迭代方向。
11. 可迁移到 AI PM 的能力。

### AI Workflow Map

建议增加一张总流程图，证明项目不是零散作品：

```text
用户场景 / 业务问题
-> 需求拆解
-> 输入信息整理
-> Prompt / 工具 / 模型选择
-> AI 生成或检索
-> 人工筛选与校验
-> 质量评估
-> 输出整理
-> bad case 复盘
-> 下一轮迭代
```

### 能力迁移表

| 原经验 | 迁移到 AI 产品的能力 |
|---|---|
| 行业场景理解 | 用户场景分析 |
| 方案推演 | 产品方案设计 |
| 多角色协作 | 跨职能沟通 |
| 内容/视觉表达 | 原型与产品叙事 |
| 专业判断 | AI 输出质量筛选 |
| 项目交付 | 需求到交付闭环 |

### AI 能力边界声明

边界声明会加分，因为它显示真实和风险意识。

```text
当前项目主要是个人产品原型与工作流探索，重点在场景拆解、Prompt 实验、内容结构化、原型表达和质量评估；尚未包装为商业上线项目或完整自动化系统。
```

## 个人网站优化

### 推荐导航

```text
Home
AI PM Cases
Workflow
Domain Proof
Resume
Contact
```

### 页面职责

| 页面 | 作用 | 重点 |
|---|---|---|
| Home | 30 秒完成求职定位 | 身份、目标岗位、核心案例、CTA |
| AI PM Cases | 展示核心 AI 产品案例 | 用户、问题、MVP、AI 链路、指标、风险 |
| Workflow | 证明 AI 工作流设计能力 | 输入、处理、输出、评估、兜底 |
| Domain Proof | 证明原行业迁移价值 | 复杂项目、专业判断、协作、交付 |
| Resume | 简历、技能、经历、下载入口 | PDF、关键词、项目摘要 |
| Contact | 投递转化 | 联系方式、目标岗位、可沟通时间 |

### 首页必须回答

```text
我是谁？
我投什么岗位？
我为什么可信？
我有哪些案例？
HR 下一步该点哪里？
```

推荐首页顺序：

1. 首屏定位 + CTA。
2. 2-3 个核心 AI 产品案例。
3. AI Workflow 能力摘要。
4. 原行业背景如何迁移为产品能力。
5. 技能关键词。
6. 简历下载 / 联系方式。

### 暂时不做

- 不重构技术栈。
- 不新增 CMS。
- 不做复杂动画。
- 不新增很多项目。
- 不把所有页面都精装修。
- 不为了“更像 AI”硬塞术语。

当前最值钱的是内容结构和求职证据，不是工程复杂度。

## 验收清单

- [ ] 首屏身份是目标 AI 产品方向。
- [ ] 简历有 2-3 个 AI 产品/Workflow 项目。
- [ ] 每个核心项目有用户、痛点、方案、AI 链路、验证、迭代。
- [ ] 原行业经历被翻译成产品能力，而不是原岗位自证。
- [ ] 作品集不只展示结果，也展示过程和评估。
- [ ] 网站首页 30 秒内能完成 HR 初筛。
- [ ] 没有夸大商业上线、真实用户、完整 Agent、算法能力或资深 PM 经验。
- [ ] 所有个人信息、联系方式、公司名、项目私密信息在公开版本中已匿名或占位。
